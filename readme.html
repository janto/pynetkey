pynetkey

This program was not written by IT and is not supported by them!
http://dip.sun.ac.za/~janto/pynetkey
For help or bug reports email Janto Dreijer at jantod@gmail.com

Pynetkey's main goals are to be more robust and prettier (it sits in the system tray) than the existing inetkey/sinetkey.

Windows

  1. download pynetkey.zip
  2. extract somewhere and run pynetkey.exe
  3. (optional) create a shortcut to pynetkey.exe on your desktop

building from source

See the Ubuntu section to get the latest sources. Run build.bat to create a distribution file.

Build requirements:
  • python2.5
  • py2exe
  • pywin32 (210 works. popup menu icons disappeared for 211,212,213)

Ubuntu

A minimal non-gui version is also available as cli.py

To install the GUI version:
  1. Install some packages: 
    sudo apt-get install python-gtk2 python-gnome2-extras
  2. Download the source. There are two ways:
    • download pynetkey.tar.gz
    • get the bleeding edge sources from the repository 
      svn co checkout: http://svn.assembla.com/svn/pynetkey
      for this to work you will need to set up the firewall in your ~/.subversion/servers file 
      [global]
http-proxy-exceptions = *.sun.ac.za
http-proxy-host = proxy.sun.ac.za
http-proxy-port = 3128
      This is a bit of a catch-22 situation, since you already need something like inetkey to get to assembla.com. Sorry about that.
  3. run pynetkey.py
  4. (optional) create a shortcut to pynetkey.py (you can use icons/main.ico as an icon)

Settings

To avoid prompting for the username and password every time you start pynetkey, you can place it in a config file.

WARNING: Please be aware how dangerous this is. If you don't know why this is dangerous, I suggest you rather not do it. Someone can easily steal your password. This is also why I won't make this feature easily available from the GUI. IT won't like the idea of people storing their passwords on disk.

Location (or just select "Edit config file..." from the popup menu):
  • On Windows: ~\inetkey.ini (i.e. C:\Documents and Settings\username\inetkey.ini)
  • On Linux: ~/.inetkeyrc

You can also force open and close events at scheduled times. This is useful for things like late night downloads / torrents. The current implementation is accurate to within 30 seconds. Will fix if I ever have time.

Example: 

[config]
username=12345678
password=supersecret

[events]
open = 01:55
close = 07:55

The password will automatically be encoded (not encrypted) to prevent wandering eyes from stealing your password.
