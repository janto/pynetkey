Pynetkey

This program was not written by IT and is not supported by them!
Pynetkey's main goals are to be more robust and prettier (it sits in the system tray) than the existing inetkey/sinetkey. It also runs on both Windows and Linux systems. Pynetkey is released under the GPL.

Main local site: http://dip.sun.ac.za/~janto/pynetkey
Bugs and feature requests go here http://bitbucket.org/janto/pynetkey/issues
If you need help, email me (Janto Dreijer) at jantod@gmail.com

Windows Installation

  1. download pynetkey.zip from the main site
  2. extract somewhere and run pynetkey.exe
  3. (optional) create a shortcut to pynetkey.exe (such as on your desktop)

building from source

See the Ubuntu section to get the latest sources. Run build.bat to create a distribution file.

Build requirements:
  • python2.5
  • py2exe
  • pywin32 (210 works. popup menu icons disappeared for 211,212,213)

Ubuntu Installation

The GUI version was developed with an ubuntu/gnome system in mind. A minimal non-gui version is also available as cli.py and should run on most systems.

To install the GUI version:
  1. Install some packages: 
    sudo aptitude install python-eggtrayicon
  2. Download the source. There are three ways:
    • download pynetkey.tar.gz from the main site
    • get the bleeding edge sources from the main repository (for this to work you need to be already connected to the internet): 
      hg clone http://bitbucket.org/janto/pynetkey/
    • get the source from a local repository mirror (might be slightly outdated): 
      hg clone static-http://dip.sun.ac.za/~janto/pynetkey/repo pynetkey
  3. run pynetkey.py
  4. (optional) create a shortcut to pynetkey.py (you can use icons/main.ico as an icon)

Config File

Disclaimer

WARNING: Please be aware how dangerous it is to store your username and password on your computer. If you don't know why this is dangerous, I suggest you rather not do it. Someone can easily steal your password. This is also why I won't make this feature easily available from the GUI. IT won't like the idea of people storing their passwords on disk.

Location

Pynetkey's default behaviour can be modified by creating a config file in the following location (or just select "Edit config file..." from the popup menu):

  • On Windows: ~\inetkey.ini (i.e. C:\Documents and Settings\username\inetkey.ini)
  • On Linux: ~/.inetkeyrc

Settings

  • Pynetkey will automatically encode the password to prevent wandering eyes from stealing your password, but it is NOT the same as it being encrypted.
  • Setting open_on_launch=1 (the default) will open the firewall on launch. Set open_on_launch=0 to disable this feature.
  • If you're on Stellenbosch Community Network it might help setting connection_url to https://fw4.sun.ac.za:950 or fw1 or fw2...
  • You can also force an open and close event at scheduled times. This is useful for things like late night downloads / torrents. The current implementation is accurate to within 30 seconds.

Example

[config]
username = 12345678
password = supersecret
open_on_launch = 1
connection_url = https://fw.sun.ac.za:950

[events]
open = 01:55
close = 07:55
